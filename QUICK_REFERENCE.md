# Correlation & Stacking Quick Reference

## Core Concepts

### üéØ Game Stack
**Definition:** Multiple players from the same game

**Why:** If a game goes over the projected total, multiple players benefit

**Example:** Tatum (BOS) + Davis (LAL) from BOS@LAL game

**Sweet Spot:** 3-4 players from one high-scoring game

---

### üèÄ Team Stack  
**Definition:** Multiple players from the same team

**Why:** When a star explodes, teammates get more touches/minutes

**Example:** Giannis + Lillard (both MIL)

**Classic Combo:** High-priced star + mid-priced role player

---

### üîÑ Bring-Back
**Definition:** Opposing player paired with a team stack

**Why:** Hedges your bet while maintaining game correlation

**Example:** 
- Stack: Tatum + Brown (BOS)
- Bring-back: Anthony Davis (LAL)

**Result:** Exposed to both sides of a high-scoring game

---

### üìä Correlation Score
**Definition:** Measure of how connected your lineup is (0-100)

**Scoring:**
- 0-30: Weak/no correlation
- 30-60: Moderate correlation  
- 60-80: Strong correlation
- 80+: Maximum correlation

**Calculation Factors:**
- Players per game
- Players per team
- Game quality (total proj)
- Number of different games

## Contest-Specific Strategies

### üí∞ Cash Games (50/50, Double-Up)
```
Correlation: 0.2 (Low)
Strategy: Floor over ceiling
Stacking: Light team stacks only
Ownership: Play the chalk
```

**Why low correlation?**
- Don't need huge ceiling
- Avoid correlated downside
- Want safety and consistency

---

### üé≤ Single Entry GPP
```
Correlation: 0.5 (Moderate)
Strategy: Balance floor and ceiling
Stacking: 1-2 mini stacks
Ownership: Mix chalk and value
```

**Balanced approach:**
- Need some ceiling to win
- Can't be too risky (only 1 lineup)
- Use correlation but don't force it

---

### üé∞ 3-Max / 20-Max
```
Correlation: 0.6-0.75 (High)
Strategy: Differentiation via stacking
Stacking: Full game stacks
Ownership: Target low-owned games
```

**Key insight:**
- Multiple lineups = can take more risk
- Want different stack exposures
- Prioritize unique structures

---

### üèÜ 150-Max (Milly Maker)
```
Correlation: 0.8-0.9 (Maximum)
Strategy: Ceiling > everything
Stacking: Multi-game stacks + bring-backs
Ownership: Contrarian when possible
```

**Tournament theory:**
- Top 0.1% of field wins big
- Need maximum differentiation
- Correlation amplifies upside
- Accept higher variance

## The Stack Spectrum

### No Correlation (0.0-0.2)
```
Lineup: 8 best value plays from 8 different games
Pro: Highest raw projection
Con: No upside leverage
Use: Cash games only
```

### Light Correlation (0.3-0.4)  
```
Lineup: 1 team stack (2 players)
Pro: Some upside, still safe
Con: Limited ceiling
Use: Conservative GPP approach
```

### Moderate Correlation (0.5-0.6)
```
Lineup: 1-2 team stacks, maybe a bring-back
Pro: Balanced risk/reward
Con: May not differentiate enough for large fields
Use: SE, small-field GPPs
```

### High Correlation (0.7-0.8)
```
Lineup: Full game stack (3-4 players from 1-2 games)
Pro: Strong upside potential
Con: Boom or bust
Use: 20-Max, medium GPPs
```

### Maximum Correlation (0.85-1.0)
```
Lineup: Multi-game stacks, bring-backs, everything connected
Pro: Massive ceiling if hits
Con: Very high variance
Use: Milly Maker, contrarian plays
```

## Game Selection Guide

### ‚úÖ Target Games
- High projected total (230+)
- Close spread (<7 points)
- Fast pace teams
- Lower game stack ownership
- Star players with value teammates

### ‚ùå Avoid Games  
- Likely blowouts (spread >12)
- Low totals (<215)
- High game stack ownership (60%+)
- Thin supporting casts
- Injury concerns

---

## Stack Building Priorities

### Priority Order:
1. **Game environment** - Start with a high-upside game
2. **Primary player** - Elite scoring/ceiling player
3. **Secondary player** - Value from same team
4. **Bring-back** - Counter-exposure from opponent  
5. **Fill spots** - Best remaining value

### Anti-Priorities (Don't):
- ‚ùå Force stacks in low-scoring games
- ‚ùå Stack teams in blowouts
- ‚ùå Over-stack (4+ from one team is risky)
- ‚ùå Ignore salary constraints for correlation

## Common Mistakes

### üö´ Mistake #1: "All or Nothing"
**Wrong:** Always use maximum correlation
**Right:** Scale to contest type

### üö´ Mistake #2: "Stack the Chalk"  
**Wrong:** Stack BOS@LAL because it's the highest total
**Right:** Stack BOS@LAL if ownership is reasonable, otherwise pivot

### üö´ Mistake #3: "Ignore Projections"
**Wrong:** Build pure stacks regardless of projection
**Right:** Balance correlation with raw projection

### üö´ Mistake #4: "Over-Stack"
**Wrong:** 5+ players from one game
**Right:** 3-4 max, spread across 3-4 games

### üö´ Mistake #5: "Lock Everything"
**Wrong:** Lock 6 players from 6 different games, then wonder why correlation doesn't work
**Right:** Lock 1-2 key pieces, let builder find the stack

## Advanced Tactics

### Contrarian Stacking
```
If BOS@LAL is 60% game stack:
‚Üí Fade it completely
‚Üí Stack DEN@PHX at 20%
‚Üí Same correlation strategy, different game
‚Üí Massive leverage if DEN@PHX hits
```

### Double Game Stack
```
Game 1: BOS@LAL (2 from BOS, 1 from LAL)
Game 2: DEN@PHX (2 from DEN)
Fills: 3 players from 3 different games
‚Üí Exposed to 2 high-upside games
‚Üí Unique structure
```

### Leverage Stack
```
Start with high-leverage player (elite ceiling, low own)
‚Üí Stack their teammates
‚Üí Add bring-back
‚Üí Entire stack is leverage play
```

## Remember

> "In GPPs, you don't win by having the highest projection.  
> You win by having a unique structure that hits."

**Correlation creates the structure. Projection still matters. Balance both.**

## Builder Tips

- Start with contest defaults

- Mix correlation levels across lineups

‚ùå **DON'T:**
